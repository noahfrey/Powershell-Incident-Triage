PowerShell Incident Triage — Mini Project
=========================================

Overview  
This project demonstrates a realistic triage workflow for a suspicious PowerShell execution event. It includes synthetic Windows Event Logs, an incident analysis, and a customer-facing summary similar to what a Tier 1 SOC or MDR analyst would produce.

What This Project Shows  
- Ability to analyze Windows security logs  
- Understanding of PowerShell execution patterns  
- Ability to distinguish suspicious behavior from benign activity  
- Clear written communication for both technical and non-technical audiences  
- Familiarity with incident documentation workflows  

Included Files  
- logs/sample-events.txt — Synthetic Windows Event Logs (Event ID 4688 and 4104)  
- incident-report.md — Full triage analysis and customer-facing summary  
- README.md — Project overview and context
   

Why This Matters

This project demonstrates core SOC skills: interpreting logs, assessing severity, and communicating findings clearly. These are foundational abilities for roles in security operations and incident response.
